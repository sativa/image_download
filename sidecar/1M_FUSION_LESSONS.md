# 1m 高分融合 · 耕地分类 — 经验与结论

> 会话日期 2026-05-31。任务:把 1m 高分影像(Esri+Google 瓦片)接进 10m 哨兵多时相耕地二分类(耕地+园地 vs 其他),
> 验证能否突破 10m 天花板。评估口径:耕地类 F1(headline)+ 准确率 + IoU。
> 数据:甘肃 89 县;跨省独立验证 = 山西长治。GPU = `ps@10.147.19.250`(4×4090),数据 store = `.174`。

---

## 一、科学结论(最重要)

### 1. 诚实基线
- 旧引用的 0.86 是**同县空间留出**(测试县也在训练县里 → 乐观)。重建真·跨县(77 训练县 / 12 留出县,120 测试 cell):
  **纯 10m 集成 @20k = F1 0.853 / 准确率 0.906 / IoU 0.73**。这≈10m + DLTB 标签的任务天花板(FTW SOTA pixel-IoU 0.76 ≈ F1 0.86)。

### 2. 1m 怎么用、值多少(跨县 / 同域)
| 方式 | 跨县 F1 |
|---|---|
| 纯 10m baseline(@5k 单模型) | 0.842 |
| route-c:1m 降采样成 2 通道提示 + 10m 光谱,**在 10m 预测** | 0.851(+0.009) |
| **route_a:1m 原生 15ch(6ch 1m + 9ch 10m 上采样),在 1m 预测** | **0.859**(1m-F1)/ 0.854(10m 聚合) |
- **1m 原生预测是最佳用法**(0.859 > 10m 网格 0.851);但**封顶 ~0.86**——到 1m 后瓶颈不再是分辨率,而是
  **DLTB 地籍边界与影像的逐像素对齐精度**。`stage-1` 1m 边界网本身 1m-F1=0.838。
- **0.9 的 F1 在当前标签质量下不可达**;0.9 的**准确率**早已达成(0.906)。

### 3. 🏆 最重磅:1m 的真正价值在跨域泛化
长治跨省(山西,160 cells):
| 模型 | argmax(0.5,部署点) | best-thr(调阈值上界) |
|---|---|---|
| 纯 10m | **0.236(崩溃)** | 0.617 |
| route-c + 1m(dual) | **0.644** | 0.670 |
| route-c + 1m(esri) | 0.613 | 0.669 |
| route-c + 1m(google) | 0.637 | 0.666 |
- 纯 10m 跨省在默认阈值**崩溃**(光谱过拟合甘肃源域,召回坍塌)→ 加 1m 救回 **+0.41(argmax)/ +0.05(调阈值)**。
- 机理:**1m 纹理/边界是"域不变"的**(地块几何跨省相似),而光谱跨省漂移 → 1m 提供跨域稳健的几何线索。
- **这是论文最强的立论点:1m 不是刷同域精度,而是让模型跨省可用。**

### 4. 三个反直觉发现(都要写进论文以免审稿人质疑)
- **更多训练数据会缩小 1m 的优势,不是放大。** 学习曲线(route-c − baseline CV F1):1k +0.035 → 2.5k +0.014 →
  5k +0.009 → 外推 20k ≈ 0。10m 模型有了足够数据就自己学到了 1m 提供的大部分信息。**所以不必为 1m 路线扩到 20k。**
- **双源(Esri+Google)不优于单源——分场景:** 跨县 esri 单源最好(dual 0.851 / esri 0.857 / google 0.853,dual 无优势);
  **跨省 dual 最优**(0.644 > google 0.637 > esri 0.613)。结论:同域单源够用,**跨域用双源更稳**(多样性抗域偏移)。
- **误差集中在小地块。** route-c 完全漏检(recall<0.1)= 14.8%(个数)/ 仅 7.2%(面积);1m 比纯 10m 多救回
  ~3pp 小地块(完全漏检 17.5%→14.8%)。这解释了为什么 1m 在面积主导的像素 F1 上只 +0.009——好处都在小地块。

### 模型规模
- stage-2:unet/b5 11ch=31.2M,unet++/b5=31.9M,deeplabv3+/b5=29.5M;route_a 15ch=31.2M。编码器 b5 占 ~28M。

## 二、实验设计经验(怎么做才"诚实/可信")

1. **先抓数据泄漏**:同县留出 → 真·跨县(留出整县)。这一步把 0.86 拉回 0.853 并暴露了真实天花板。
2. **干净消融 = 唯一变量**:route-c vs baseline 用 `--no-1m` 把 1m 的 2 通道**置零**(同架构/同种子/同 5k 数据),
   差值才是纯 1m 贡献。源对照用 `--sources esri|google|both` 切 x6 通道(同样只变源)。
3. **同口径对照**:v40_5k 与 v40_xcounty 的**测试集完全相同**(同 120 cell / 12 县),只训练量不同 → 可比。
4. **多口径评估**:像素 F1(面积主导)会掩盖小地块问题 → 必须加**对象级漏检率**(`polygon_miss.py`)+ **1m 原生 F1**
   (小地块按数量计权)。同一模型在不同口径下结论不同。
5. **跨省是必做的硬测试**:同域结论(1m 小用)会误导;跨省才暴露 1m 的真价值(+0.41)。**域偏移下务必报 argmax(部署点)
   和 best-thr(上界)两个数**——baseline 跨省 argmax 0.236 vs thr 0.617,只报一个会严重误导。
6. **学习曲线 > 单点**:想知道"加数据会不会更好",训 1k/2.5k/5k 看趋势 + 外推,比硬训 20k 便宜得多(且本例外推结论=不值得)。

## 三、工程经验(踩过的坑 + 解法)

### 传输(关键)
- **Mac↔服务器走 ZeroTier,总带宽封顶 ~8MB/s**(不是单流限制——4 流并行实测仍 8MB/s)。光纤只在服务器之间(192.168.x),
  Mac 不在网内。服务器直连/proxy 抓瓦片仅 ~4.5KB/s,**无法在服务器侧重下**。
- 解法:**数据必过这条慢链路,就把字节砍掉**——164GB RGBA-tif 重压成 **16GB JPEG-GeoTIFF(10.3×)**,ship 8h→33min。
- 教训:遇到慢传先**实测**(单流/多流吞吐 + 各链路 + 重下可行性),别假设。

### JPEG-GeoTIFF(纯 Rust 手写,有坑)
- `tiff` crate(0.10)encoder **只支持 无压缩/LZW/Deflate,不支持 JPEG**;LZW 对高熵影像几乎无效(1.3×);
  Deflate+predictor 也只 ~2.3×。要 10× 必须 JPEG(有损)。
- 手写 JPEG-in-TIFF 的正确形式(= GDAL 的形式):**photometric=YCbCr(6) + JPEGTables(347, 共享量化/Huffman表) +
  缩略流 strip + RefBlackWhite(532) + YCbCrSubSampling(530)**。坑:
  - 完整 JFIF 当单 strip 塞进去 → libtiff 报 `zero scanline size`(失败)。
  - photometric=RGB(2) → 能读但**颜色错**(YCbCr 当 RGB 读,MSE~1500)。
  - 缺 532 RefBlackWhite → photometric=6 也读不了。
  - `image` crate 出的是 4:2:2 YCbCr JFIF;做 marker 拆分得到 tables + 缩略流即可。**从 SOF 读子采样设 530**(自动匹配)。
- 质量:q92 PSNR 40dB、q95 43dB,边界保真(梯度相关 0.99,强边缘 IoU 0.86–0.90)→ **q92–q95 对分割/边界无实质损害**
  (源本就是 JPEG 瓦片,再压一次可忽略)。行业标准就是 YCbCr-JPEG-GeoTIFF(GDAL/QGIS/提供商)。
- 已落地为下载器 CLI 默认输出:`--compress jpeg|deflate|none`(默认 jpeg),提交 `0d75f36`,74 测试过、worktree 独立构建验证。

### GPU 编排(踩坑)
- **stage-1 推理是 IO 密集 + GPU 轻载**(加载 29MB/cell 的 npz),别按 GPU 利用率判断进度;**也别假设它慢**——
  本例误判"还要 77min"其实快完了,导致手动放任务上 GPU2/3 差点撞上消融 stage-2。
- 教训:**用 poll 式调度器(launch 前查 GPU 真实空闲),别硬编码 GPU + 别基于时间假设**;复位用脚本(setsid + `</dev/null`),
  别在 ssh 里内联 `nohup ... &`(输出会截断、状态难判)。
- 多个后台 follow + 重复启动会产生**冗余重跑**;关键结果一旦 final.json 落地就该停掉重跑省 GPU。

### 各种小坑
- macOS `rsync` 不认 `--info=progress2`;`cat` 不认 `-A`。
- ssh heredoc 里 f-string 转义引号会炸 → 用 `%` 格式化。
- `torch.load(..., weights_only=True)`;`pickle.load` 自建文件加注释说明可信来源。

## 四、天花板与下一步

- **同域天花板 ~0.86 F1**,由 **DLTB 标签↔影像对齐精度**决定(不是分辨率、不是模型)。想破 0.9 得提升标签:
  更精细/更新的地类边界、或对影像重新数字化勾绘、或半监督/目标域微调。
- **跨省**才是 1m 的主战场,也是论文价值所在;可继续:(a) 更多省份验证泛化;(b) 跨省微调/域适应把 0.64 往上推;
  (c) route_a(1m 原生)+ 双源在跨省的组合(本次跨省只测了 route-c)。
- **路线 b(双支路后融合)**仍未测(记录待办)。
